A console-based ATM Simulation built using C programming language.

This project demonstrates structured programming concepts including:
- `struct` usage
- Pointer-based function design
- Authentication logic
- Transaction tracking system
- Menu-driven interface
- ANSI color formatting in terminal

## Features
* ✔ Multi-user login system (PIN authentication)
* ✔ 3 login attempts limit
* ✔ Check account balance
* ✔ Deposit money
* ✔ Withdraw money
* ✔ Transaction history tracking
* ✔ Individual balance per user
* ✔ Clean terminal UI with color formatting

## Concepts Used
- Structures (`typedef struct`)
- Arrays inside structures
- Pointer to struct (`Account *user`)
- Menu-driven program using loops
- Conditional logic
- Transaction logging using signed values
- Basic system design thinking

## Account Structure
Each user account contains:
```c
typedef struct {
    char name[50];
    int pin;
    float balance;
    float transactions[1000];
    int transactionCount;
} Account;
```
## Transaction Logic
- Deposits are stored as positive values
- Withdrawals are stored as negative values
- Transaction history is generated by checking value sign
- Example:
`[500, -200, 100]`

Means:
- Deposited $500
- Withdrew $200
- Deposited $100

## Menu Options
- Check Balance
- Deposit
- Withdraw
- View Transactions
- Exit

## 🔐 Authentication System
- Users must enter correct PIN
- Maximum 3 failed attempts allowed
- Program exits after 3 incorrect attempts

## How to Compile & Run
Using GCC:
```bash
gcc atm_simulation.c -o atm
./atm
```
